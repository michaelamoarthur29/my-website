Overview

Your Personal Fitness Companion

ADIM Workouts is an all-in-one fitness platform that combines AI coaching, nutrition planning, social community, and analytics into a single web app. It's designed to feel like a real consumer product — polished UI, real-time data, and personalized experiences — all powered by Python, Streamlit, and Google's cloud stack.

The app serves two types of users: goal-oriented athletes who want structured plans and data-driven insights, and casual users who want smart, effortless recommendations. ADIM handles both with a single adaptive interface.

Technical Documentation

Project Architecture

The app follows a clean three-layer architecture — separating data access, display logic, and routing into distinct, testable modules.

Project File Structure
adim-workouts/
├── app.py — entry point, tab routing, custom CSS
├── modules.py — all UI display functions
├── data_fetcher.py — BigQuery reads/writes + Gemini AI calls
├── home_page.py — homepage slideshow component
├── internals.py — HTML component templating engine
├── modules_test.py — unit tests for display functions
└── data_fetcher_test.py — mocked BigQuery + AI tests

data_fetcher.py — Data Layer

  • Reads user profiles, workouts, and posts from BigQuery
  • Calls Gemini AI for context-aware workout and nutrition advice
  • Writes new posts and leaderboard entries back to BigQuery
  • Returns clean, structured dicts consumed by modules
  • All external calls mocked in tests for reliability

modules.py — Display Layer

  • display_post() — social feed card with profile image + caption
  • display_activity_summary() — Altair chart analytics dashboard
  • display_genai_advice() — multi-turn AI chat with session state
  • display_nutritional_ai_advice() — 7-day meal plan generator
  • display_leaderboard() — streak tracking + community rankings

app.py — Routing Layer

  • Six Streamlit tabs: Home, Leaderboard, Community, AI Chat, My Workouts, Your Posts
  • Custom CSS overrides for tab positioning, heading styles, spacing
  • Session state management across Streamlit reruns
  • display_activity_page() helper isolates workout tab logic

internals.py — Templating

  • Loads HTML files from custom_components/ directory
  • Replaces {{key}} placeholders with safe-escaped data
  • Renders via Streamlit's components.html() for rich UI elements
  • Handles quote/backslash escaping to prevent injection
Features

Six Modules. One App.

🤖 AI Workout Coaching

Users select experience level (Beginner → Pro) and goals, then chat with "John" — a Gemini-powered AI assistant. Full multi-turn conversation with session-state history persistence across Streamlit reruns.

🥗 AI Meal Planning

Generates a personalized 7-day meal plan from allergies, nutrition goals, weekly budget, and cooking time. Returns full shopping list (organized in 3 columns), estimated cost, and daily macro summary.

📊 Activity Analytics

Altair-powered bar charts comparing steps and calories across workout sessions. Four metric tiles (total workouts, distance, steps, calories) plus a side-by-side workout comparison view.

👥 Community Feed

Social feed of friends' workout posts ordered by timestamp. Each post shows profile picture, username, content, and post image. AI-generated daily motivation quote shown at the top of the feed.

🏆 Leaderboard

Tracks workout streaks, favorite exercises, and weekly plans. Daily streak confirmation modal creates a re-engagement habit loop. Users see their community rank and top performers.

📤 Progress Sharing

One-click "Post My Steps" button auto-generates a social post from the most recent workout — lowering friction for sharing milestones and driving community engagement.

Engineering Notes

Key Decisions

The AI chat interface uses session state to persist conversation history across Streamlit reruns — a non-trivial problem in Streamlit's stateless execution model. The assistant response only fires when the last message role is "user", preventing duplicate responses on rerun.

modules.py · display_genai_advice()

Custom CSS overrides moved the tab navigation to the top-right corner of the viewport — giving the main content area maximum breathing room while keeping all six tabs accessible without a sidebar.

app.py · Streamlit CSS overrides

The homepage uses a time-based auto-cycling slideshow built in pure Streamlit — tracking elapsed time in session state and triggering st.rerun() to advance slides, since Streamlit has no native timer component.

home_page.py · create_streamlit_slideshow()
Testing Strategy

How It's Tested

modules_test.py

Unit tests for every display function — verifying correct rendering paths, edge cases (missing username → "Anonymous", missing image → default URL), and error handling for invalid inputs like null timestamps.

data_fetcher_test.py

All BigQuery reads/writes and Gemini AI calls are mocked using unittest.mock.patch — isolating tests from live cloud services and ensuring consistent, repeatable results regardless of environment.
